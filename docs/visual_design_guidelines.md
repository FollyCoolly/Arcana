# Reality Mod - 视觉设计注意事项（P5 Inspired）

> **版本**: v0.1.0  
> **最后更新**: 2026-02-10  
> **状态**: Draft

---

## 1. 文档目标

本文档用于约束 Reality Mod 的视觉实现，目标是形成 **Persona 5 风格启发** 的界面语言，同时保持原创和可维护性。

- 借鉴重点：高对比、斜切几何、强节奏排版、分层动效
- 非目标：直接复刻游戏素材、Logo、角色、UI 贴图

---

## 2. 参考项目观察

本规范基于以下开源项目的可见实现特征整理：

- https://github.com/LzxHahaha/persona5
- https://github.com/caffi-nate/Persona5-Weather-Forecast
- https://github.com/TingTyphoon/persona5-shibuya-station
- https://github.com/q-mona/p5-ui
- https://github.com/Mive82/Persona-5-Calendar

归纳出的共性：

- 主色集中在 `红 / 黑 / 白`，辅色使用非常克制
- 倾斜与切角是核心视觉语法（而非圆角卡片）
- 通过 CSS 或 SVG 做层叠、遮挡、错位，制造速度感
- 文本排版比图形更强势（大字号、粗字重、标签化文案）

## 3. 色彩使用规范

### 3.1 颜色角色（Design Tokens）

```css
:root {
  --rm-bg-0: #09090b;      /* 页面主背景 */
  --rm-bg-1: #14141a;      /* 次级背景层 */
  --rm-surface: #f6f6f3;   /* 亮面卡片 */
  --rm-text-0: #f4f4f5;    /* 深色背景上的主文字 */
  --rm-text-1: #151518;    /* 浅色背景上的主文字 */
  --rm-primary: #e6122f;   /* 主强调红 */
  --rm-primary-strong: #ff2647; /* 高亮红 */
  --rm-accent: #ffd233;    /* 警示/提示黄 */
  --rm-muted: #8b8b95;     /* 次级信息 */
  --rm-success: #26c281;   /* 成功态 */
  --rm-danger: #ff4d4f;    /* 危险态 */
}
```

### 3.2 配色比例

- `背景色（黑/深灰）`: 60% - 75%
- `中性色（白/浅灰）`: 15% - 30%
- `强调红`: 8% - 12%
- `功能色（黄/绿等）`: <= 5%

### 3.3 使用规则

- 一个组件内最多使用 `1` 个主强调色（默认红）。
- 同一屏出现多个红色块时，必须有至少 `1` 个浅色或深色缓冲区。
- 红色优先用于“操作焦点/状态高亮”，不要用于大段正文背景。
- 文本对比度最低达到 `WCAG AA`（正常文本 >= 4.5:1）。

## 4. 排版与字体规范

### 4.1 字体角色

- 标题（英数）: `Bebas Neue`, `Archivo Black`, `Impact`（后备）
- 正文（中文）: `Noto Sans SC`, `Source Han Sans SC`, `Microsoft YaHei`
- 数值（可选）: `Rajdhani`, `Barlow Condensed`

### 4.2 字号层级

- `Display`: 48-64px（模块标题）
- `H1`: 32-40px
- `H2`: 24-28px
- `Body`: 14-16px
- `Meta`: 11-12px（标签、注释）

### 4.3 排版规则

- 大标题可使用全大写英文字母与轻微负字距。
- 标签文本优先短词，不写长句。
- 数据数字必须在视觉上比标签至少高 `1` 个层级。

---

## 5. 形状与布局规范

### 5.1 几何语言

- 主斜切角只允许使用以下角度：`6deg / 8deg / 12deg`
- 一个页面最多并存 `2` 种斜切角，避免混乱
- 圆角只用于次级元素，核心卡片优先硬边或切角

### 5.2 分层原则

- 至少 3 层：`背景层`、`结构层`、`交互层`
- 使用偏移和投影表达层级，不依赖纯边框
- 可使用伪元素 `::before/::after` 做斜切遮罩与底板

### 5.3 信息密度

- 首屏优先“高密度概览”，减少无意义留白
- 关键指标固定在视觉热区（左上到中部）
- 模块之间用斜切分隔，而非普通横线

---

## 6. 组件风格约束

### 6.1 按钮

- 主按钮：红底黑字或白字，支持轻微倾斜
- 次按钮：深底白边，悬停时出现红色切片高亮
- 不使用默认圆角 pill 按钮样式

### 6.2 卡片

- 卡片需带“底板层”（阴影或错位副层）
- 标题区与数据区有明显分区
- 状态条/角标可使用黄色作为次强调

### 6.3 导航与标题条

- 导航项使用短标签（1-2 词）
- 当前项使用红色 + 形状变化（不仅仅变色）
- 页面标题应具备“标识性轮廓”（斜切底板或描边）

---

## 7. 动效规范

### 7.1 时长与曲线

- `fast`: 120ms（hover/focus）
- `base`: 180ms（切换/按钮）
- `slow`: 260ms（模块进场）
- 推荐曲线：`cubic-bezier(0.2, 0.8, 0.2, 1)`

### 7.2 动效类型

- 进场：`位移 + 透明度`，位移建议 8-16px
- 切换：遮罩擦入或斜切片扫过
- 强调：短促闪烁或计数跳动，不做长时间循环动画

### 7.3 限制项

- 禁止超过 `400ms` 的主流程过渡
- 禁止多个大型动画并发触发
- 支持 `prefers-reduced-motion` 降级

---

## 8. 纹理与素材边界

### 8.1 推荐素材

- 轻噪点纹理（CSS/SVG 程序化生成）
- 几何矢量块、速度线、切片遮罩
- 自绘图标或开源图标库二次统一风格

### 8.2 禁止事项（版权与品牌边界）

- 不直接使用 Persona 官方角色、UI 截图、Logo、字体文件
- 不使用可识别的官方图形资产做商业或公开发布
- 仅借鉴“设计语言”，所有视觉资产需原创或合规授权

---

## 9. 落地优先级（Reality Mod）

1. 在 `src/app.css` 建立颜色与排版 tokens
2. 重构 `src/routes/+page.svelte` 的首屏层级与斜切结构
3. 抽离通用组件：`SectionHeader`、`StatCard`、`SkewButton`
4. 补充动效变量，统一过渡时长与曲线
5. 最后再处理纹理和装饰，不先做高成本视觉细节

---

## 10. 验收清单

- 是否在不使用官方素材的前提下仍能体现风格特征
- 是否形成稳定 token，而不是页面级临时样式
- 是否保证可读性和对比度，不因风格牺牲信息可读
- 是否在桌面与移动尺寸下保持相同视觉语法
