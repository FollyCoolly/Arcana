# Reality Mod - 视觉风格指南（P5-Inspired）

> **最后更新**: 2026-03-07
> **状态**: Active Draft

---

## 1. 文档目标

本文档用于约束 Reality Mod 的视觉实现，目标是形成 **受 Persona 5 启发（P5-inspired）** 的界面语言，同时保持原创和可维护性。

- 借鉴重点：高对比、斜切几何、强节奏排版、分层动效
- 非目标：复刻官方素材、角色、Logo、截图、贴图

---

## 2. 品牌与版权边界

版权保护与是否盈利无关。关键区分如下：

**禁止（无论盈利与否）：**
- 使用官方游戏字体文件、贴图、截图、角色图形、Logo
- 直接复制特定 UI 截图的像素级还原

**完全允许：**
- 深度学习并原创实现 P5 的设计语言（斜切几何、红黑白对比、强节奏排版）
- 设计风格、构图逻辑、色彩关系本身不受版权保护

**结论：** 本项目应积极、深入地学习 P5 的设计语言并原创实现，而不是因误解版权边界而自我限制。

---

## 3. 核心视觉语法

- 主色基调严格限定为 `红 / 黑 / 白`
- 窗口可采用透明背景，界面主体以不规则图形层叠呈现
- 倾斜与切角是核心语法，避免通篇圆角卡片化
- 通过层叠、遮挡、错位制造节奏和速度感
- 文本层级优先，标题与数据应比装饰图形更强势
- **UI 元素必须有强形状**：裸文字不构成 UI 组件，任何按钮、标签、导航项都需要有几何底座
- **状态变化必须是形状+颜色双重变化**，不允许仅改变透明度或颜色来表达交互状态

---

## 4. 色彩规范

### 4.1 颜色角色（Design Tokens）

```css
:root {
  --rm-black: #000000;
  --rm-white: #ffffff;
  --rm-red: #80001a;
}
```

### 4.2 使用规则

- **永远不要使用渐变（linear-gradient / radial-gradient / conic-gradient 全部禁止）**
- 配色只允许 `红 / 黑 / 白` 三色体系，不引入黄/蓝/绿/紫等第四种颜色
- 每种颜色只允许 `1` 个固定值，不允许 `soft/strong/deep/dim` 等变体 token
- 单组件最多 `1` 个主强调色（默认红）
- 红色用于”操作焦点/状态高亮”，UI 组件层避免大面积正文红底
- **全局蒙版层例外**：界面呼出时整屏背景为深红色蒙版（见第 6 节），此为底层沉浸效果，不属于 UI 组件层
- 同一屏 UI 红色块较多时，必须通过黑白块面与留白做缓冲
- 文本对比度至少满足 `WCAG AA`（正常文本 >= 4.5:1）

---

## 5. 排版与字体规范

### 5.1 字体角色

- 标题（英数）：窄体、粗体、高识别度字形
- 正文（中文）：可读性优先的无衬线字体
- 数值（可选）：偏工业感或紧凑字形

字体栈（按优先级降级）：

```css
font-family: "p5hatty", "Orbitron", Arial, sans-serif;
```

- 首选：`p5hatty`（项目自定义字体）
- 备选：`Orbitron`（几何感等宽风格）
- 回退：`Arial`, `sans-serif`

### 5.2 字号层级

- `Display`：菜单主项，`clamp(3rem, 7vw, 4.8rem)`（约 48–77px）
- `H1`: 32-40px
- `H2`: 24-28px
- `Body`: 14-16px
- `Meta`: 11-12px（标签、注释）

### 5.3 排版规则

- 大标题可用全大写英文字母并配轻微负字距
- 标签优先短词，不写长句
- 关键数据在视觉上至少高于标签 `1` 个层级

---

## 6. 形状与布局规范

### 6.1 几何语言

- 主斜切角建议在 `6deg-18deg` 区间
- 单页可并存 `2-4` 种斜切角，但需有主次关系
- 圆角仅用于次级元素，核心区优先硬边或切角

### 6.2 分层原则

界面呼出时为 4 层结构（从底到顶）：

| 层级 | 名称      | 实现方式                                 | 说明                                                               |
| ---- | --------- | ---------------------------------------- | ------------------------------------------------------------------ |
| 0    | OS 模糊层 | Tauri `Effect::Acrylic`                  | 将桌面及其他窗口磨砂模糊                                           |
| 1    | 蒙版层    | CSS `background: rgba(150, 0, 15, 0.80)` | 全屏深红色半透明覆盖，营造游戏沉浸感                               |
| 2    | 结构层    | 几何背景图形（星形等）                   | 7 层同心五角星（黑白交替，`scale` 0.92→0.20），尺寸 80vh，居于左侧 |
| 3    | 交互层    | 菜单项、信息面板                         | 可点击的 UI 组件                                                   |

其他规则：

- 用偏移和投影表达层级，避免仅靠边框分隔
- 可用 `::before/::after` 构建斜切遮罩与底板

### 6.3 信息密度

- 首屏优先高密度概览，减少无意义留白
- 关键指标放在视觉热区（左上至中部）
- 模块分隔优先斜切语法，不用普通横线

## 7. 组件风格约束

### 7.1 按钮

- **主按钮**：红底白字，带斜切形状（`clip-path`），hover 时形状扩张或翻转为黑底
- **次按钮/功能按钮**：黑底白字，带斜切形状，hover 时切换为红底
- **返回/导航按钮**：独立几何色块（非裸文字），见 7.4
- 禁止：透明背景裸文字按钮、圆角 pill 按钮、仅靠 opacity 表达 hover 状态
- 箭头符号使用几何风格（CSS/SVG 实现），不使用 Unicode 箭头字符（`←→`）

### 7.2 卡片

- 卡片需有底板层（阴影或错位副层）
- 标题区与数据区有明确分区
- 状态条/角标仅使用红或白，不引入第四色

### 7.3 导航与标题条

- 导航项使用短标签（1-2 词）
- 当前项需”颜色 + 形状”双重变化，不只变色
- 菜单项形状为**梯形**：左侧（靠近星星）高度为右侧的 70%，`clip-path: polygon(0 15%, 100% 0%, 100% 100%, 0 85%)`
- 菜单项旋转角度呈**放射状**：顶部 −9° 均匀递进至底部 +9°，模拟从左侧星星向外辐射的扇形
- 奇偶项保持横向错位（`translateX`），叠加旋转角度形成节奏感
- 页面标题应具备可识别轮廓（斜切底板或描边）

### 7.4 返回按钮

P5 中的返回/导航控件从不是裸文字，必须是独立的几何块面：

- **形状**：斜切矩形（`clip-path` 左侧或右侧切角），与标题栏形成对位关系
- **配色**：默认黑底白字；hover 变为红底白字
- **箭头**：CSS border 或 SVG 实现的几何三角箭头，不使用 `←` 字符
- **位置**：嵌入标题导航栏左侧，作为独立色块，与页面标题色块形成并排或错位关系
- **尺寸**：高度与同行标题块持平，不小于标题块高度的 80%

```
┌─────────────────────────────────────┐
│ [▶ BACK]  [   STATUS   ]            │  ← 两个独立几何块，BACK 为斜切色块
└─────────────────────────────────────┘
```

## 8. 动效规范

### 8.1 时长与曲线

- `fast`: 120ms（hover/focus）
- `base`: 180ms（切换/按钮）
- `slow`: 260ms（模块进场）
- 推荐曲线：`cubic-bezier(0.2, 0.8, 0.2, 1)`

### 8.2 动效类型

- 进场：`位移 + 透明度`，位移建议 8-16px
- 切换：遮罩擦入或斜切片扫过
- 强调：短促闪烁或计数跳动，避免长循环动画

### 8.3 限制项

- 主流程过渡不超过 `400ms`
- 避免多个大型动画并发触发
- 必须支持 `prefers-reduced-motion` 降级

---

## 9. 验收清单

- 是否在不使用官方素材前提下仍体现风格特征
- 是否形成稳定 token，而非页面级临时样式
- 是否保证可读性与对比度，不以风格牺牲信息传达
- 是否在桌面与移动尺寸下保持同一视觉语法
- 是否避免”普通矩形窗口感”，实现透明背景下的非规则主体构图
- **所有按钮是否有几何底座**，是否消除了所有”裸文字 + opacity hover”按钮
- **返回按钮**是否为独立几何色块，是否与标题块形成并排对位关系
